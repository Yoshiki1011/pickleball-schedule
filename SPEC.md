# ピックルボール紅白対抗戦スケジュール生成 仕様書

## 1. 基本構成

- **対戦形式**: ダブルス（2人 vs 2人）
- **チーム**: 紅組 vs 白組（各組同人数）
- **コート**: 複数面同時進行
- **1試合 = 1ラウンド**（全コートが同時に行われる）

## 2. 人数・出場・休みの制約

### 2.1 出場人数
- 1試合あたりの出場人数 = コート数 × 4人
- 1試合あたりの休み人数 = 総人数 - 出場人数
- 片組の休み人数 = 休み人数 / 2（整数でなければエラー）

### 2.2 均等出場
- 全員が同じ試合数だけ出場する
- 1人あたり出場数 = (試合数 × 出場人数) / 総人数（整数でなければ警告）

### 2.3 休みの分散
- 休みは全員に均等に分散させる
- 連続して休まないよう配慮する

## 3. 対戦モード

### 3.1 制限なし（any）
- 性別を考慮しない
- 任意の2人がペアを組める
- 任意のペア同士が対戦できる

### 3.2 同性対戦（same）
- **ペア**: 同性同士のみ（男男ペア or 女女ペア）
- **対戦**: 同性ペア同士のみ（男男 vs 男男、女女 vs 女女）
- **必須条件**: 各試合で出場する男性数・女性数がそれぞれ偶数であること

### 3.3 ミックス対戦（mixed）
- **ペア**: 男女混合のみ（男1人 + 女1人）
- **対戦**: ミックスペア同士のみ（男女 vs 男女）
- **必須条件**: 各試合で出場する男性数 = 女性数であること

### 3.4 自動（混合）モード（auto）
- 同性対戦とミックス対戦を組み合わせる
- ミックス割合スライダーで比率を調整（0%=同性のみ、100%=ミックスのみ）
- **各試合の対戦タイプは出場男女数によって決まる**（事前決定ではない）

#### autoモードの対戦タイプ決定ルール
1. 出場男女数を確認（紅組・白組それぞれ）
2. **ミックス可能条件**: 紅組の出場男性数 = 出場女性数 かつ 白組の出場男性数 = 出場女性数
3. **同性可能条件**: 紅組の出場男性数・女性数がともに偶数 かつ 白組の出場男性数・女性数がともに偶数
4. ミックス割合に基づいて優先度を決定:
   - ミックス可能 かつ (ミックス優先 or 同性不可) → ミックス試合
   - 同性可能 → 同性試合
   - どちらも不可 → anyモード（フォールバック）

#### ⚠️ 重要: 不正な対戦の禁止
- **autoモードでは「男男 vs 女女」「ミックス vs 男男」などの混在は絶対に発生させない**
- 各コートの対戦は必ず同じ性別タイプ同士（男男vs男男、女女vs女女、男女vs男女）

## 4. 休みスケジュール生成（generateRestSchedule）

### 4.1 性別制約あり（same/mixed/autoモード）

#### 有効な(restM, restF)の組み合わせを列挙
- restM: 男性の休み人数、restF: 女性の休み人数
- restM + restF = restPerGame（片組の休み人数）
- **sameモード/autoモード**: 出場男性数(playM)と出場女性数(playF)がともに偶数
- **mixedモード**: 出場男性数(playM) = 出場女性数(playF)

#### 候補選択ルール
- **性別制約ありの候補のみを使用**（制約なしは廃止）
- 各候補で「休んだ後の全員の休み回数の分散」を計算
- 分散が最小になる候補を選択
- 性別制約ありの候補が存在しない場合のみ「制約なし」にフォールバック

#### ⚠️ 重要: 「制約なし」候補の廃止理由
- 「制約なし」で男1人・女1人を選ぶと出場が男奇数・女奇数になる可能性がある
- 奇数人数ではペアが作れず、不正な対戦（ミックス vs 男男など）が発生する

### 4.2 性別制約なし（anyモード）
- 全員から休み回数が少ない順に選ぶ

## 5. ペア生成（assignPartners）

### 5.1 ペアの有効性
- **anyモード**: 任意の2人
- **sameモード**: 同性同士のみ
- **mixedモード**: 異性同士のみ
- **autoモード**: 任意の2人（対戦時に制御）

### 5.2 同一ペア最大回数
- 同じ人と組める最大回数（デフォルト2回）
- ミックス対戦は+1自動加算（男女ペアは組み合わせが少ないため）

## 6. コート割り当て（assignCourts）

### 6.1 対戦の有効性チェック（isMatchValid）
- **anyモード**: 任意
- **sameモード**: 同性ペア同士（男男vs男男、女女vs女女）
- **mixedモード**: ミックスペア同士（男女vs男女）
- **autoモード**: 同じ性別タイプ同士（rGenders === wGenders）

### 6.2 スコアリング
- 対戦相手の重複を最小化
- 完全重複（4人全員が既に対戦済み）は大きなペナルティ

## 7. 検証（validate）

### 7.1 エラー条件（生成不可）
- 紅組と白組の人数が異なる
- コート数 ≥ 片組人数
- 片組の休み人数が整数でない
- 同性モードで出場男女数が偶数にならない
- ミックスモードで出場男女数が同数にならない

### 7.2 警告条件（生成可能だが注意）
- 1人あたり出場数が整数でない（不均等）
- autoモードで出場男女数が偶数にならない場合がある

## 8. 実装チェックリスト

### generateRestSchedule
- [ ] sameモード/autoモード: playM%2===0 && playF%2===0 の組み合わせのみ列挙
- [ ] mixedモード: playM===playF の組み合わせのみ列挙
- [ ] 性別制約ありの候補の中から分散最小を選ぶ
- [ ] 性別制約ありの候補がない場合のみフォールバック

### generateSchedule（autoモード）
- [ ] 各試合の出場男女数を確認してgameModeを動的決定
- [ ] canMixed: 紅組playM===playF かつ 白組playM===playF
- [ ] canSame: 紅組playM%2===0&&playF%2===0 かつ 白組playM%2===0&&playF%2===0
- [ ] ミックス割合に基づいて優先度を決定

### assignCourts
- [ ] autoモードでisMatchValid: rGenders===wGenders のみ許可
- [ ] 制約を満たす組み合わせがない場合のみフォールバック
